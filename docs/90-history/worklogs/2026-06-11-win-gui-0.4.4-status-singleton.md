# 2026-06-11 Windows GUI 0.4.4 状态刷新与单例修正

## 背景

用户反馈 0.4.3 后速度明显提升,但 GUI 仍有几个产品问题:

- 版本号应提升到 `0.4.4`,并显示在 GUI 上。
- 未换 IP 时,UI 仍会周期性刷新公网 IP,偶尔显示「获取中...」。
- IPv4 与 IPv6 希望同时展示。
- 客户端需要保持单例排他。

## 改动

- 版本号:
  - `clients/desktop-gui/package.json`
  - `clients/desktop-gui/package-lock.json`
  - `clients/desktop-gui/src-tauri/Cargo.toml`
  - `clients/desktop-gui/src-tauri/Cargo.lock`
  - `clients/desktop-gui/src-tauri/tauri.conf.json`
  全部提升到 `0.4.4`。
  - Tauri 新增 `app_version()` 命令,前端在主界面底部显示 `v0.4.4`。
- 状态刷新:
  - CLI `status` 新增 `--no-ip-check` 参数。
  - Tauri `status()` 改为调用 `zhvpn status --json --no-ip-check`,只刷新本地运行状态/本地代理/出口名。
  - 新增 Tauri `status_ip()` 专门执行公网 IP 探测。
  - 前端只在连接成功、首次进入主界面、换 IP 后、以及约 60s 低频周期调用 `status_ip()`。
  - 前端保留上一轮有效 IPv4/IPv6,避免短暂探测空窗把 UI 刷成「获取中...」。
- 双栈展示:
  - `netcheck.PublicIPsViaHTTPProxy` 分别探测 IPv6 endpoint 与 IPv4 endpoint。
  - `status --json` 新增 `egress_ipv6` / `egress_ipv4` 字段;旧 `egress_ip` 保持兼容,优先返回 IPv6。
  - GUI 改为两行显示「出口 IPv6」「出口 IPv4」。
- 单例排他:
  - GUI 主进程使用 Windows 命名 Mutex `Local\ZonghengVPNDesktop`;重复打开第二个 GUI 进程会直接退出。
  - CLI `start` / `stop` / `login` / `import` / `logout` 在同一 `ZHVPN_HOME` 下使用 Windows 命名 Mutex `Local\ZonghengVPN-<home-hash>`,避免并发操作同时启动/切换同一实例。

## 说明

公网出口 IP 是外部观察结果,本机代理进程无法从本地网卡直接可靠知道 NAT/代理链路最终暴露给外网的地址,所以仍需要请求公网 IP endpoint。0.4.4 的变化是不再把公网 IP 探测放进高频状态轮询里,并且把 IPv4 与 IPv6 分开展示。

后续产品边界已修正:Hub 不应作为出口兜底。v4-only endpoint 若无法经手机 IPv4 返回,应在 GUI 中如实表现为 IPv4 不可用,不能把 Hub VPS `36.50.84.68` 当作出口展示。
