# 2026-06-10 工作记录：GUI 0.3.1 x64 客户包重打包

## 背景

客户在 Win10 i5 机器上打开上一版 GUI 安装器时提示：

```text
此应用无法在你的电脑上运行
若要找到适用于你的电脑的版本，请咨询软件发布者。
```

此前同一客户可运行 CLI `zhvpn.exe`，因此客户机器大概率是 Windows x64，问题更可能出在 GUI 安装器/打包链路，而不是 VPN 核心 CLI。

## 处理

- 版本号提升到 `0.3.1`，避免客户继续混用旧 `0.3.0` 包。
- 在 macOS 上补齐交叉构建工具链：Rust stable、`x86_64-pc-windows-msvc` target、`cargo-xwin`、LLVM、NSIS。
- `clients/desktop-gui/build.ps1` 增加 `-Runner auto|cargo|cargo-xwin`，非 Windows host 默认使用 `cargo-xwin` 构建 Windows target。
- amd64 sidecar 构建强制 `GOAMD64=v1`，保证老 Intel i5 / Win10 兼容。
- 重新生成客户安装器，并复制到 `dist/windows-amd64/`：
  - `ZonghengVPN-0.3.1-windows-x64-setup-VERIFIED.exe`
  - `ZonghengVPN-0.3.1-windows-x64-setup-VERIFIED.zip`

## 验证

构建命令实际执行：

```bash
GOOS=windows GOARCH=amd64 GOAMD64=v1 go build -tags with_gvisor -trimpath -ldflags "-s -w" -o src-tauri/binaries/zhvpn-x86_64-pc-windows-msvc.exe ../cli
npm install
npm run tauri build -- --target x86_64-pc-windows-msvc --runner cargo-xwin
```

PE header 校验：

- 安装器 `纵横 VPN_0.3.1_x64-setup.exe`：machine `0x014c`，NSIS x86 installer，可在 x64 Windows 上运行。
- 主程序 `zhvpn-desktop.exe`：machine `0x8664`，x64/amd64。
- sidecar `zhvpn.exe`：machine `0x8664`，x64/amd64。

SHA256：

- 安装器：`507fd6f71daba95033836942cdaaa55b60ac8e0512de3196bf8dda45a4e19917`
- ZIP：`fd7db7dcbd96e6d89d547d91b00230b4475c24924cbe72720f3c4b6925609b28`

ZIP 完整性：Python `zipfile.testzip()` 返回 OK。

## 注意

- 当前包未代码签名，客户机器可能出现 SmartScreen。若提示“Windows 已保护你的电脑”，需点“更多信息”→“仍要运行”。
- WebView2 仍使用 `downloadBootstrapper`。如果客户可运行安装器但安装后 GUI 无法打开，再考虑改 `offlineInstaller` 发大包。
